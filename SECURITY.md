# Security Policy

## Supported Versions

The latest minor of the latest major receives security updates.

| Version | Supported |
| ------- | --------- |
| 1.x     | ✅        |
| < 1.0   | ❌        |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security problems.

Email **tamishmhatre26@gmail.com** with:

- A description of the vulnerability and its impact.
- Steps to reproduce or a proof-of-concept.
- Any suggested mitigation.

We will acknowledge receipt within 72 hours and aim to publish a fix within 14 days for critical issues. Once a patch is released we'll credit you in the changelog (unless you prefer to remain anonymous).
